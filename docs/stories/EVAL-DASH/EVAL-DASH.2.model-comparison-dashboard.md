# Story EVAL-DASH.2: Model Comparison Dashboard

## Status

Ready for Review

## Story

**As a** site visitor (recruiter, hiring manager, or peer),
**I want** a visual dashboard comparing spaCy vs Claude entity extraction with precision/recall/F1 charts and per-entity-type breakdowns,
**so that** I can immediately see the quantitative evaluation results and understand each extractor's strengths.

## Acceptance Criteria

| # | Criterion |
|---|-----------|
| AC1 | Headline F1 cards render for both extractors with prominent, colored values |
| AC2 | Grouped bar chart shows P/R/F1 by government branch with side-by-side extractor comparison |
| AC3 | Per-entity-type table shows TP/FP/FN breakdown for each extractor |
| AC4 | Branch filter/tabs (All / Legislative / Executive / Judicial / CoNLL) control which data is displayed |

### Data Source: Evaluation Result Files

**Location:** `eval/reports/baseline/` (relative to repo root, one level up from `frontend/`)

**`summary.json`** structure:
```json
{
  "generated": "2026-03-25",
  "branches": {
    "judicial": {
      "spaCy en_core_web_sm": { "precision": 0.192, "recall": 0.925, "f1": 0.318, "true_positives": 74, "false_positives": 311, "false_negatives": 6, "article_count": 15 },
      "Claude Sonnet": { "precision": 0.456, "recall": 0.938, "f1": 0.614, "true_positives": 75.5, "false_positives": 90, "false_negatives": 5, "article_count": 15 }
    },
    "executive": { ... },
    "legislative": { ... },
    "conll": { ... }
  }
}
```

**Per-branch `*_results.json`** structure (Promptfoo output):
```json
{
  "evalId": "eval-...",
  "results": {
    "prompts": [
      {
        "provider": "spaCy en_core_web_sm",
        "metrics": {
          "namedScores": {
            "true_positives": 74, "false_positives": 311, "false_negatives": 6,
            "person_tp": 27, "person_fp": 137, "person_fn": 0,
            "government_org_tp": 17, "government_org_fp": 62, "government_org_fn": 4,
            "organization_tp": 0.5, "organization_fp": 41, "organization_fn": 0,
            "location_tp": 13, "location_fp": 18, "location_fn": 1,
            "event_tp": 0, "event_fp": 0, "event_fn": 1,
            "concept_tp": 16.5, "concept_fp": 52, "concept_fn": 0,
            "legislation_tp": 0, "legislation_fp": 1, "legislation_fn": 0,
            "Precision": 0.192, "Recall": 0.925, "F1": 0.318,
            "extracted_count": 386, "gold_count": 81
          }
        }
      },
      {
        "provider": "Claude Sonnet",
        "metrics": {
          "namedScores": { ... same structure ... }
        }
      }
    ]
  }
}
```

**Key data notes:**
- TP/FP/FN values can be fractional (e.g., `0.5`) due to fuzzy matching partial credit
- Entity type fields follow pattern: `{entity_type}_{metric}` (e.g., `person_tp`, `government_org_fp`)
- 7 entity types: person, government_org, organization, location, event, concept, legislation
- `Precision`/`Recall`/`F1` (capitalized) are the article-level metrics; lowercase `precision`/`recall`/`f1` are Promptfoo's internal scaled scores — use the **capitalized** versions
- Per-branch files are ~200KB each (Promptfoo output with per-test-case results); only extract the `namedScores` from each prompt's metrics

### Next.js API Route Pattern

The project does not yet have custom API routes beyond the Web Vitals proxy at `app/api/vitals/route.ts`. Follow this pattern:

```typescript
// app/api/eval/results/route.ts
import { NextResponse } from 'next/server';
import fs from 'fs';
import path from 'path';

export async function GET() {
  const filePath = path.join(process.cwd(), '..', 'eval', 'reports', 'baseline', 'summary.json');
  const data = JSON.parse(fs.readFileSync(filePath, 'utf-8'));
  return NextResponse.json(data);
}
```

**Important:** `process.cwd()` in Next.js resolves to the `frontend/` directory. The eval results are one level up (`../eval/reports/baseline/`).

```typescript
import { useQuery } from '@tanstack/react-query';
import { getEvalSummary } from '@/lib/api/evaluation';

export function useEvalSummary() {
  return useQuery({
    queryKey: ['eval-summary'],
    queryFn: getEvalSummary,
    staleTime: 24 * 60 * 60 * 1000, // 24 hours — data changes only on new eval runs
  });
}
```

### Weighted F1 Calculation

To compute an overall F1 across branches for the headline cards, weight by article count:

```
overall_f1 = Σ(branch_f1 × branch_article_count) / Σ(branch_article_count)
```

For "All" view, include all 4 branches. For gov-domain only, exclude CoNLL.
